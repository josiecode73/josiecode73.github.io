Glassmorphism 2026：10 個 App 以後，我總算弄清楚什麼時候不該用它

J
JosieCode73 · 2026 年 1 月 · 6 min read

Glassmorphism 這個詞大概在 2021 年左右開始流行，伴隨著 macOS Big Sur 的視覺語言。四五年後的現在，它沒有消失，但用的人更懂得克制了——至少我是在把十個 app 全部做過一遍以後，才真正懂得克制。

這篇不是「如何實作毛玻璃效果」的教學，那種文章到處都是。這篇是關於什麼時候不該用，以及用的時候要注意哪些實際問題。

backdrop-filter 在做什麼（從瀏覽器角度）

理解效能問題之前，先理解瀏覽器在做什麼。backdrop-filter 的工作是把元素背後的內容做模糊處理。聽起來簡單，但這件事在渲染流程裡很貴：

  1. 瀏覽器需要把「背後的內容」先渲染到一個獨立的 layer（合成層）
  2. 對這個 layer 做 blur 計算（GPU 計算，blur 半徑越大越貴）
  3. 把模糊後的結果合成到最終畫面

這意味著 backdrop-filter 會強制建立一個新的 compositing layer。如果頁面上有很多這樣的元素，記憶體消耗和 GPU 負擔都會顯著上升。

blur 值選多少才合理

我自己的實測結果（M2 MacBook + Chrome）：

  • blur(4px)：幾乎感受不到效能影響
  • blur(8-16px)：視覺效果好，效能可接受的甜蜜區間
  • blur(32px) 以上：在低端設備上開始掉幀

實際的 glass card CSS 大概長這樣：

.glass-card {
  background: rgba(255, 255, 255, 0.12);
  backdrop-filter: blur(12px) saturate(180%);
  -webkit-backdrop-filter: blur(12px) saturate(180%);
  border: 1px solid rgba(255, 255, 255, 0.2);
  border-radius: 16px;

  /* 重要：幫助瀏覽器優化 */
  transform: translateZ(0);
  will-change: auto;    /* 不要預設 will-change: transform，除非這個元素真的會動 */
}
⚠️ 不要在 scroll 容器裡放大量 glass 元素。每次滾動，瀏覽器需要重新計算每個 glass 元素背後的模糊，這是掉幀的常見原因。

Safari 的那些坑

Safari 對 backdrop-filter 的支援不算差，但有幾個讓人抓狂的例外：

在 SVG 背景上失效

如果你的背景是 SVG（比如內聯 SVG 或 SVG data URL），Safari 的 backdrop-filter 可能完全不工作。這個 bug 在 Safari 17 以後有改善，但還是要測試。

在 position: fixed 元素上的問題

固定定位的 nav bar 加 backdrop-filter，在 iOS Safari 上滾動時偶爾會出現閃爍。解法是確保 nav bar 的 z-index 設置合理，並且它的父元素沒有 transform 或 filter 屬性（這會打破 containing block）。

-webkit- prefix 還是需要

截至 2026 年，-webkit-backdrop-filter 在 Safari 上仍然是必要的。不加的話某些舊版 Safari 完全沒有效果。

Glass UI 的前提條件：背景必須有足夠的對比

這個道理簡單，但很多人忘了：毛玻璃效果在純色背景上沒有意義。如果背景是 #ffffff，玻璃卡片放上去只是一個帶模糊的白色方塊，看不出任何「透明感」。

Glass UI 需要背景有：顏色變化、漸層、圖案、或者移動的內容（比如影片背景）。在我的 dark theme app 裡，背景用了帶角度的漸層配合一些低透明度的幾何圖形，這才讓玻璃效果有了意義。

✓ 適合 Glass UI 的情境

  • Dark mode 儀表板（背景有漸層深度）
  • 浮動的 modal / tooltip
  • 固定 nav bar（背景內容在其下方滾動）
  • 短暫出現的 toast 通知

✗ 不適合的情境

  • 亮色系純白背景
  • 長列表 / 大量卡片（效能問題）
  • 低端設備優先的應用
  • 需要高對比度的無障礙場景

一個讓我改變想法的觀察

在做 FlowBoard（看板工具）的時候，我把所有卡片都做成 glass 效果，覺得很美。然後讓幾個朋友用了一下，收到的回饋是：「卡片跟背景的邊界不清楚，有點難判斷哪個是卡片」。

後來我把一般卡片改回純色，只保留 modal 和 nav bar 的 glass，整體可用性反而更好。

Glass UI 是個強調「氛圍」的設計語言，不是強調「資訊」的。如果你的 UI 需要用戶快速判斷、點擊、做決策，清晰的邊界比漂亮的毛玻璃更重要。

2026 年的現實：它已經是 baseline

backdrop-filter 的瀏覽器支援率在 2026 年已經超過 95%。你不再需要為「要不要用」這個問題糾結太久，更值得思考的是「在哪裡用、用多少」。

我的最終建議是：把 glass 效果保留給浮在內容上方的元素（nav、modal、dropdown），不要把它用在作為內容容器的元素（文章卡片、資料列表）。這個原則讓效能問題消失的同時，可用性也更好。
